# Чеклист запуска ArtStudio CMS в продакшен

Пошаговый список для вывода сайта на боевой хостинг. Выполняйте по порядку;
каждый пункт можно отметить `[x]`.

## 1. Развёртывание

- [ ] Залить код на сервер (git clone или архив). Вебрут (DocumentRoot/root)
      должен указывать на каталог **`public/`**, а не на корень проекта.
- [ ] Apache: убедиться, что работает `.htaccess` (`AllowOverride All`,
      включён `mod_rewrite`). Nginx: применить `docs/nginx.conf.example`.
      При ошибке 403 — раздел «403 Forbidden» в `docs/DEPLOY.md`.
- [ ] Права: `storage/` и `public/uploads/` доступны на запись PHP-процессу;
      `config/` и `storage/` находятся ВНЕ вебрута (при вебруте `public/` —
      так и есть).
- [ ] Создать БД MySQL/MariaDB (utf8mb4) и пользователя с правами на неё.
- [ ] Открыть сайт в браузере → автоматически запустится установщик
      `/install`: проверка окружения → подключение БД → данные сайта →
      супер-админ.
- [ ] PHP 8.2+ с расширениями: pdo_mysql, gd, zip, curl, mbstring, dom.

## 2. Окружение (переменные / config.php)

- [ ] `APP_URL` — боевой домен с https (важно: используется в sitemap,
      каноникалах, письмах и Open Data).
- [ ] `APP_ENV=production`, `APP_DEBUG=false`.
- [ ] SMTP: `SMTP_HOST/PORT/USERNAME/PASSWORD/ENCRYPTION`, `MAIL_FROM_*` —
      без этого не работают сброс пароля, email-уведомления и дайджест.
- [ ] `BACKUP_EXTERNAL_DIR` — каталог ВНЕ сервера (смонтированное сетевое
      хранилище / rclone / S3-FUSE). Локальная копия бэкапа бесполезна при
      полном отказе сервера.
- [ ] По желанию: `TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID` — алерты об ошибках
      из логов (это отдельный канал от бота для входа, см. п. 4).

## 3. Cron (обязательно)

Без cron не работают: очередь писем, вебхуки, автопостинг, автобэкапы,
дайджест, GDPR-очистка. Скопировать из `docs/DEPLOY.md`:

```
* * * * *  php /path/app/Console/mail_worker.php     >> /path/storage/logs/mail_worker.log 2>&1
* * * * *  php /path/app/Console/webhook_worker.php  >> /path/storage/logs/webhook_worker.log 2>&1
*/5 * * * * php /path/app/Console/social_worker.php   >> /path/storage/logs/social_worker.log 2>&1
0 3 * * *  php /path/app/Console/backup_worker.php    >> /path/storage/logs/backup_worker.log 2>&1
30 3 * * * php /path/app/Console/gdpr_cleanup.php     >> /path/storage/logs/gdpr_cleanup.log 2>&1
0 9 * * 1  php /path/app/Console/digest_worker.php    >> /path/storage/logs/digest_worker.log 2>&1
```

- [ ] Cron установлен; `/health` возвращает `ok` (а не `degraded`).

## 4. Telegram-бот (вход в панель + уведомления)

- [ ] Создать бота у **@BotFather** (`/newbot`, бесплатно), токен вставить в
      «Настройки → Вход в панель: код через Telegram».
- [ ] Каждому администратору привязать Telegram в «Профиле» (отправить боту
      одноразовый код) — вход будет подтверждаться 6-значным кодом.
- [ ] chat_id администраторов вписать в «Настройки → Уведомления о заявках»
      — заявки с форм и сбои автобэкапа будут приходить в Telegram.
- [ ] Проверить вход: выйти и войти заново — должен прийти код от бота.

## 5. Бэкапы (проверить ДО запуска)

- [ ] Дождаться первого ночного бэкапа (или запустить вручную:
      `php app/Console/backup_worker.php`).
- [ ] Убедиться, что копия появилась в `BACKUP_EXTERNAL_DIR`.
- [ ] **Один раз прогнать восстановление** на тестовую БД:
      `php database/restore.php <архив.zip> <тестовая_БД> <каталог>` —
      непроверенный бэкап равен отсутствию бэкапа.

## 6. Переезд со старого сайта (asdr.gov.uz)

- [ ] Перенести контент: страницы, новости, документы, вакансии, тендеры
      (структура и типы контента уже готовы).
- [ ] Собрать список старых URL (выгрузка из старой CMS / Я.Вебмастер /
      Search Console) и импортировать в «Редиректы → Массовый импорт»
      (формат: `/старый-путь /новый-путь` построчно).
- [ ] После запуска раз в неделю смотреть «Редиректы → Недавние 404» —
      живые старые ссылки закрывать кнопкой «Создать 301».
- [ ] Отправить `https://домен/sitemap.xml` в Я.Вебмастер / Search Console.
- [ ] Для data.gov.uz: передать эндпоинты `/opendata` (JSON, кэш 24 ч).

## 7. Контент и внешний вид

- [ ] «Настройки»: название сайта, контакты (телефон/email/адрес — попадают
      в футер и Schema.org), логотип, favicon.
- [ ] «Дизайн»: выбрать палитру/пресет (живое превью — до сохранения);
      рекомендуется сохранить стартовый вид как «Мою конфигурацию».
- [ ] «Меню»: разделы навигации (дублируются в футере).
- [ ] Страницы: сохранить главную как шаблон («Шаблоны страницы») — легко
      вернуть исходный вид.
- [ ] Удалить демо-контент, если сидился (или заменить реальным).

## 8. Безопасность перед открытием

- [ ] Сменить пароль супер-админа с установочного; включить Telegram-код.
- [ ] Завести редакторов с ролью `editor` (без доступа к системным разделам).
- [ ] Проверить, что `/install` отвечает 404 (лок установки на месте).
- [ ] HTTPS: сертификат установлен, http → https редирект на уровне сервера.
- [ ] Проверить `/robots.txt` и `/sitemap.xml` на боевом домене.

## 9. Контроль после запуска (первая неделя)

- [ ] `/health` в мониторинг (uptime-сервис) — `degraded` значит встал воркер.
- [ ] «Журнал действий» — активность администраторов.
- [ ] «Редиректы → Недавние 404» — добить потерянные ссылки.
- [ ] `storage/logs/error.log` — должен быть пустым или почти пустым.
