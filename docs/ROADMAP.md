## 1. Второй релиз — функциональность

### 1.1. Электронная приёмная (обращения граждан) — главная фича v2

Типовое требование к сайту госоргана. Сейчас есть простые формы; нужно:

- регистрационный номер обращения (выдаётся после отправки);
- страница «Проверить статус обращения» по номеру;
- статусы в админке: новое → в работе → отвечено (+ срок по регламенту,
  подсветка просроченных);
- уведомление заявителю на e-mail при смене статуса (очередь писем уже есть);
- выгрузка реестра обращений (CSV) за период.

Фундамент готов: формы, очередь писем, капча, согласие ПДн, срок хранения ПДн.

### 1.2. Перевод контента (узбекский/английский)

Движок мультиязычности готов (раздел «Языки», локализация блоков).
Работа контентная: перевести страницы, меню, новости-якоря. Плюс код:

- [x] переключатель языков в шапке показывать только для языков
      с заполненным контентом (`Locale::contentLangs`);
- [x] hreflang в `<head>` для переведённых страниц (только языки
      с реальным контентом, x-default = основной язык).

### 1.3. Открытые данные — расширение

`OpenDataController` уже отдаёт базовые наборы. Добавить:

- [ ] автоматические выгрузки CSV/JSON для новых типов контента
      (вакансии, тендеры, мероприятия);
- [ ] паспорт набора данных (дата обновления, периодичность, контакт).

### 1.4. Медиа

- [ ] Батч-конвертация ранее загруженных изображений в WebP
      (консольный воркер; новые загрузки уже конвертируются);
- [ ] автогенерация размеров (srcset) для картинок в блоках.

## 2. Второй релиз — безопасность (уровень 2)

Первый уровень пройден (см. отчёт в сессии: OWASP-базлайн зелёный). Дальше:

- [x] CSP на публичной части через nonce (`SecurityHeaders::publicCsp`;
      хосты GA/Метрики/Google Fonts — только при включённых настройках;
      nonce в кэшированных блоках подставляется при отдаче).
- [x] Убрать `unsafe-inline` из админ-CSP (инлайн-обработчики перенесены
      в admin.js/frontend.js: фильтры списков, тип виджета, капча).
- [x] HSTS preload — опция `security.hsts_preload` в config
      (включать вручную после месяца стабильного HTTPS).
- [x] Журнал входов с IP в «Журнале действий» (`AuditLog::auth`, метод
      AUTH: вход/ошибка/блокировка/2FA/выход).

## 3. Техдолг и качество

- [ ] E2E-смоук в CI (Playwright уже используется локально: логин в
      админку, создание страницы, отправка формы с капчей).
- [ ] Lighthouse-бюджет на проде (цель: Performance ≥ 90 на главной).
- [ ] Индексы БД: пройтись EXPLAIN'ом по спискам каталога и новостей
      при > 1000 записей.
- [ ] Документация редактора: короткая памятка «как публиковать новость»
      (с Telegram и push) для сотрудников пресс-службы.
